# iRacing Telemetry SDK - AI Agent Guide

> **For AI coding assistants**: rules and patterns for generating code using the iRacing Telemetry SDK (v1.0+).

## Minimal Example (copy-paste ready)

```csharp
using Microsoft.Extensions.Logging;
using SVappsLAB.iRacingTelemetrySDK;

// declare the telemetry variables your app needs — only these become properties on TelemetryData
[RequiredTelemetryVars([TelemetryVar.RPM])]
public class Program
{
    public static async Task Main(string[] args)
    {
        var logger = LoggerFactory.Create(b => b.AddConsole()).CreateLogger("App");

        // pass null ibtOptions for live telemetry, or new IBTOptions("file.ibt") for playback
        IBTOptions? ibtOptions = args.Length > 0 ? new IBTOptions(args[0]) : null;
        await using var client = TelemetryClient<TelemetryData>.Create(logger, ibtOptions);
        using var cts = new CancellationTokenSource();
        Console.CancelKeyPress += (_, e) => { e.Cancel = true; cts.Cancel(); };

        var counter = 0;
        var subscriptionTask = client.SubscribeToAllStreams(
            onTelemetryUpdate: async data =>
            {
                // telemetry arrives at 60Hz — throttle output to once per second
                if (counter++ % 60 != 0) return;
                Console.WriteLine($"RPM: {data.RPM ?? 0f:F0}");
            },
            onSessionInfoUpdate: async session =>
            {
                // read the current driver's name from parsed session info
                var driverIdx = session.DriverInfo?.DriverCarIdx ?? 0;
                var driverName = session.DriverInfo?.Drivers?[driverIdx]?.UserName ?? "unknown";
                Console.WriteLine($"Driver: {driverName}");
            },
            cancellationToken: cts.Token);

        await Task.WhenAny(client.Monitor(cts.Token), subscriptionTask);
    }
}
```

## Critical Rules

1. **Never manually create `TelemetryData`** — it is generated by the source generator from the `[RequiredTelemetryVars]` attribute
2. **Use `TelemetryVar` enums, not strings** — `[RequiredTelemetryVars([TelemetryVar.Speed])]` not `[RequiredTelemetryVars(["Speed"])]`
3. **All telemetry properties are nullable** (`float?`, `int?`, `bool?`) — always handle nulls
4. **Use `await using`** for client disposal — `await using var client = TelemetryClient<TelemetryData>.Create(logger);`
5. **`GetTelemetryVariables()` is synchronous** — do not await it; returns `IReadOnlyList<TelemetryVariable>`
6. **Keep stream readers lightweight** — telemetry arrives at 60Hz; heavy work causes buffer overflow and data loss
7. **Target .NET 8.0+** required
8. **Windows required for live telemetry**; IBT playback is cross-platform

## Nullable Handling

```csharp
// correct patterns
var speed = data.Speed ?? 0f;
var display = $"Speed: {data.Speed?.ToString("F1") ?? "N/A"}";
if (data.IsOnTrackCar == true) { /* explicit true check for bool? */ }
var speedMph = data.Speed * 2.23694f; // preserves null semantics
if (data.Speed.HasValue && data.Speed.Value > 100) { /* safe conditional */ }

// never do this
var speed = data.Speed.Value;          // throws if null
if (data.IsOnTrackCar) { }            // won't compile (bool? to bool)
```

## Client Creation

Two `Create()` overloads — both return `ITelemetryClient<TelemetryData>`:

```csharp
// overload 1: live telemetry (ibtOptions defaults to null)
ITelemetryClient<TelemetryData> client = TelemetryClient<TelemetryData>.Create(logger);

// overload 1: IBT file playback
var ibtOptions = new IBTOptions("file.ibt", playBackSpeedMultiplier: 1);
await using var client = TelemetryClient<TelemetryData>.Create(logger, ibtOptions);

// overload 2: with ClientOptions (requires both ibtOptions and clientOptions)
var clientOptions = new ClientOptions { MeterFactory = meterFactory };
await using var client = TelemetryClient<TelemetryData>.Create(logger, ibtOptions, clientOptions);
```

### IBT playback speed options

```csharp
new IBTOptions("file.ibt");                                // default: int.MaxValue (as fast as possible)
new IBTOptions("file.ibt", playBackSpeedMultiplier: 1);    // real-time (1x)
new IBTOptions("file.ibt", playBackSpeedMultiplier: 10);   // 10x speed
```

## Two Consumption Approaches

### Simple: SubscribeToAllStreams (recommended)

All delegates are **optional** — only provide what you need.

```csharp
[RequiredTelemetryVars([TelemetryVar.Speed, TelemetryVar.RPM, TelemetryVar.Gear])]
public class Program
{
    public static async Task Main(string[] args)
    {
        var logger = LoggerFactory.Create(b => b.AddConsole()).CreateLogger("App");
        await using var client = TelemetryClient<TelemetryData>.Create(logger);
        using var cts = new CancellationTokenSource();

        var subscriptionTask = client.SubscribeToAllStreams(
            onTelemetryUpdate: async data => Console.WriteLine($"Speed: {data.Speed}"),
            onSessionInfoUpdate: async session => Console.WriteLine($"Track: {session.WeekendInfo.TrackDisplayName}"),
            onRawSessionInfoUpdate: async yaml => Console.WriteLine($"YAML length: {yaml.Length}"),
            onConnectStateChanged: async state => Console.WriteLine($"Connection: {state}"),
            onError: async error => Console.WriteLine($"Error: {error.Message}"),
            cancellationToken: cts.Token);

        var monitorTask = client.Monitor(cts.Token);
        await Task.WhenAny(monitorTask, subscriptionTask);
    }
}
```

### Advanced: Direct Stream Access (maximum performance)

Use when you need >500K records/sec, custom backpressure, or selective consumption.

```csharp
var telemetryTask = Task.Run(async () =>
{
    await foreach (var data in client.TelemetryData.WithCancellation(cts.Token))
    {
        // process telemetry
    }
}, cts.Token);

var monitorTask = client.Monitor(cts.Token);
await Task.WhenAny(monitorTask, telemetryTask);
```

### Available Streams

| Stream | Type | Description |
|--------|------|-------------|
| `client.TelemetryData` | `IAsyncEnumerable<TelemetryData>` | 60Hz telemetry |
| `client.SessionData` | `IAsyncEnumerable<TelemetrySessionInfo>` | parsed session info |
| `client.SessionDataYaml` | `IAsyncEnumerable<string>` | raw YAML session data |
| `client.ConnectStates` | `IAsyncEnumerable<ConnectState>` | connection changes (`Connected`, `Disconnected`) |
| `client.Errors` | `IAsyncEnumerable<Exception>` | error notifications |

## Client Status and Control

```csharp
// connection status
bool connected = client.IsConnected;

// pause/resume telemetry stream writes (thread-safe, idempotent)
client.Pause();                // suppresses stream writes; internal processing continues
bool paused = client.IsPaused; // check pause state
client.Resume();               // resumes stream writes
// note: eventual consistency — 1-2 samples (~16-32ms) may pass before effect

// discover all available telemetry variables (synchronous)
IReadOnlyList<TelemetryVariable> vars = client.GetTelemetryVariables();
foreach (var v in vars)
{
    // TelemetryVariable has: Name, Type, Length, IsTimeValue, Desc, Units
    Console.WriteLine($"{v.Name}: {v.Desc} ({v.Units})");
}
```

## Buffer Behavior

All streams use a **60-sample ring buffer** (1 second at 60Hz). If consumption is slower than production, oldest unread samples are dropped. Keep processing fast or offload heavy work:

```csharp
// offload heavy work
await foreach (var data in client.TelemetryData.WithCancellation(ct))
{
    _ = Task.Run(() => PerformExpensiveAnalysis(data));
}
```

## Common Variable Categories

| Category | Variables |
|----------|-----------|
| Vehicle | `Speed` (m/s), `RPM`, `Gear`, `Throttle` (0-1), `Brake` (0-1), `Clutch`, `SteeringWheelAngle` (rad) |
| Position | `LapDistPct` (0-1), `IsOnTrack`, `IsOnTrackCar`, `PlayerTrackSurface` |
| Timing | `SessionTime`, `LapCurrentLapTime`, `LapBestLapTime`, `LapLastLapTime` |
| Safety | `PlayerIncidents` (IncidentFlags), `EngineWarnings` (flags), `SessionFlags` |
| Systems | `FuelLevel` (L), `WaterTemp` (C), `OilTemp` (C), `OilPress` (bar) |
| Environment | `AirTemp` (C), `TrackTemp` (C), `WindVel` (m/s), `TrackWetness` |
| Multi-car | `CarIdxLapDistPct` (float[]), `CarIdxPosition` (int[]), `CarIdxOnPitRoad` (bool[]) |

Use `client.GetTelemetryVariables()` to discover all 400+ available variables at runtime.

## Anti-Patterns Summary

| Don't | Do Instead |
|-------|------------|
| `[RequiredTelemetryVars(["Speed"])]` | `[RequiredTelemetryVars([TelemetryVar.Speed])]` |
| `public struct TelemetryData { ... }` | let source generator create it |
| `using var client = ...` | `await using var client = ...` |
| `data.Speed.Value` | `data.Speed ?? 0f` or `data.Speed.GetValueOrDefault()` |
| `if (data.IsOnTrackCar) {}` | `if (data.IsOnTrackCar == true) {}` |
| `await client.GetTelemetryVariables()` | `client.GetTelemetryVariables()` (synchronous) |
| `client.OnTelemetryUpdate += ...` | `client.SubscribeToAllStreams(...)` or direct streams |
| `Thread.Sleep` / `.Wait()` / `.Result` in stream | offload heavy work to `Task.Run` |
| `var client = TelemetryClient.Create(...)` | `TelemetryClient<TelemetryData>.Create(...)` |
| access undeclared telemetry properties | only properties in `[RequiredTelemetryVars]` exist |
| `new IBTOptions("nonexistent.ibt")` | validate file path before creating IBTOptions |

## Source Generator Notes

- `[RequiredTelemetryVars]` must use compile-time constant `TelemetryVar` enum arrays
- attribute targets classes only (`AllowMultiple = false`, not inherited)
- only variables declared in the attribute are available as properties on `TelemetryData`
- clean build sometimes needed after changing the attribute
- view generated code: IDE > Dependencies > Analyzers > SVappsLAB.iRacingTelemetrySDK.CodeGen

## Dependency Injection Pattern

The SDK has no built-in DI extensions. Register manually:

```csharp
services.AddSingleton<ITelemetryClient<TelemetryData>>(provider =>
{
    var logger = provider.GetRequiredService<ILogger<Program>>();
    IBTOptions? ibtOptions = args.Length == 1 ? new IBTOptions(args[0]) : null;
    return TelemetryClient<TelemetryData>.Create(logger, ibtOptions);
});
```

## Agent Checklist (before finalizing code)

- [ ] attribute uses `TelemetryVar` enums only (not strings)
- [ ] all referenced telemetry fields are declared in `[RequiredTelemetryVars]`
- [ ] client created via `TelemetryClient<TelemetryData>.Create(logger, ...)`
- [ ] client disposed via `await using`
- [ ] nullability handled for display, conditions, and math
- [ ] `Monitor()` task is started and coordinated with consumption task(s) via `Task.WhenAny`
- [ ] stream loop has no blocking or heavy work
- [ ] if using IBT playback, file path is validated before creating `IBTOptions`
- [ ] `GetTelemetryVariables()` is not awaited (it is synchronous)

## AI Decision Framework

1. **Default to SubscribeToAllStreams** unless max performance is explicitly needed
2. **Always declare variables with `TelemetryVar` enums**
3. **Always handle nullability** — use `?.`, `??`, `.HasValue`, or `== true` for bools
4. **Always use `await using`** for client lifetime
5. **Throttle console/file output** — 60Hz is too fast for unthrottled I/O
6. **Use `ITelemetryClient<TelemetryData>`** as the declared type for DI and testability
